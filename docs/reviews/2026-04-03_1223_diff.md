## Critical Findings

### C1: Spacecraft re-render storm — 60fps React reconciliation

**File**: `src/components/Spacecraft.tsx:10` + `src/hud/HUD.tsx:9`
**Agents**: Paradigm Enforcer, DS&A Reviewer
**Issue**: `Spacecraft` and `HUD` subscribe to `s.spacecraft` from Zustand. `DataDriver` calls `setSpacecraft()` every frame, creating a new object via spread (`{ ...prev.spacecraft, ...state }`). This triggers re-renders of `Spacecraft`, `HUD`, and all HUD children at ~60fps. The `Html` overlay in Spacecraft is especially expensive.
**Fix**: Use `useRef` for spacecraft position in `Spacecraft.tsx` — read from `getState()` inside `useFrame`. In `HUD`, use individual scalar selectors (`s.spacecraft.speed`, etc.) or throttle store updates to ~4Hz for display purposes.

### C2: Hot-path linear scan in interpolator — O(n) per frame

**File**: `src/data/interpolator.ts:26-34`
**Agent**: Complexity Analyst
**Issue**: `lagrangeInterpolate` performs O(n) linear scan over all vectors to find the closest point. With ~3,600 vectors (10-day mission at 4-min intervals), that's ~216,000 comparisons/second at 60fps. Vectors are already sorted by epoch.
**Fix**: Replace with binary search — reduces to ~12 comparisons per frame.

### C3: Per-frame array allocations in interpolator

**File**: `src/data/interpolator.ts:41-52`
**Agent**: Complexity Analyst
**Issue**: Each frame creates 13 temporary arrays: 1 from `vectors.slice()` (line 41), 12 from `.map()` calls inside the component loop (2 per component x 6 components). This causes GC pressure at 60fps (780 allocations/second).
**Fix**: Extract `times` array once before the component loop. Pass `startIdx`/`endIdx` instead of slicing. Hoist the components array to module scope.

### C4: Stale closure in useChat

**File**: `src/hooks/useChat.ts:28`
**Agent**: Paradigm Enforcer
**Issue**: `sendMessage` captures `messages` from closure, but by the time the API call runs, `setMessages` on line 15 has already added the user message. The closure holds the pre-update `messages`, so the API receives an incomplete history.
**Fix**: Use `useRef` to track latest messages, or build `allMessages` from functional state update.

### C5: fetchedRef guard breaks StrictMode + orphaned setTimeout

**File**: `src/hooks/useOEM.ts:11-26`, `src/hooks/useDSN.ts:11-28`
**Agent**: Paradigm Enforcer
**Issue**: The `fetchedRef.current = true` guard prevents useEffect from re-running in React StrictMode's double-mount cycle. After the first unmount clears intervals, the guard blocks re-creation on the second mount — polling silently dies. Additionally, `useOEM` line 26 creates a `setTimeout(fetchOEM, 30_000)` retry that is never cleared on unmount.
**Fix**: Remove `fetchedRef` guards. Use AbortController for fetch cancellation. Store timeout IDs in refs and clear in cleanup.

## Warning Findings

### W1: CameraController unnecessary spacecraft subscription

**File**: `src/components/CameraController.tsx:12`
**Agent**: Paradigm Enforcer
**Issue**: Subscribes to `s.spacecraft` but never uses it in render — the `useEffect` reads from `getState()`. Causes unnecessary re-renders at 60fps.
**Fix**: Remove line 12.

### W2: CameraController follow-orion target is stale

**File**: `src/components/CameraController.tsx:17-48`
**Agent**: Paradigm Enforcer
**Issue**: Target positions for `follow-orion` and `earth-view` are computed in `useEffect([cameraMode])` — only when mode changes. The camera target does not track Orion's movement over time.
**Fix**: Move target calculation for `follow-orion` into `useFrame` using `getState()`.

### W3: HUD subscribes to entire spacecraft object

**File**: `src/hud/HUD.tsx:9`
**Agent**: Paradigm Enforcer, DS&A Reviewer
**Issue**: `HUD` subscribes to `s.spacecraft` and passes individual fields as props. New object every frame causes all children to re-render at 60fps.
**Fix**: Subscribe to individual scalar fields at leaf component level, or use Zustand's `useShallow`.

### W4: DSNStatus inline computation on every render

**File**: `src/hud/DSNStatus.tsx:13-29`
**Agent**: Paradigm Enforcer
**Issue**: `stationStatuses` computed inline with `find()`/`some()` on every render.
**Fix**: Wrap in `useMemo(() => ..., [stations])`.

### W5: Trajectory past/future split is stale

**File**: `src/components/Trajectory.tsx:9-37`
**Agent**: Complexity Analyst, Paradigm Enforcer
**Issue**: `useMemo` depends only on `[oemData]`, but the past/future split uses `Date.now()`. The dividing line is frozen when OEM data loads, never moves.
**Fix**: Add a coarse timer dependency (every 60 seconds) to refresh the split.

### W6: ChatPanel uses array index as key

**File**: `src/chat/ChatPanel.tsx:60`
**Agent**: Paradigm Enforcer
**Issue**: `key={i}` for messages. Fine while messages are append-only, breaks if editing/deletion is added.
**Fix**: Add unique `id` to ChatMessage type.

### W7: useMission creates new object every second

**File**: `src/hooks/useMission.ts:8-10`
**Agent**: Paradigm Enforcer
**Issue**: `getMissionElapsed()` returns a new object every tick, causing all consumers to re-render every second.
**Fix**: Low priority. Acceptable at 1Hz for MVP.

### W8: API key in URL query parameter

**File**: `api/chat.ts:61`
**Agent**: Paradigm Enforcer
**Issue**: `${GEMINI_URL}?key=${apiKey}` puts the API key in the URL, which gets logged in server/proxy access logs.
**Fix**: Use `x-goog-api-key` header instead.

### W9: Wildcard CORS on chat endpoint

**File**: `api/chat.ts:31`
**Agent**: Paradigm Enforcer
**Issue**: `Access-Control-Allow-Origin: *` allows any website to use the Gemini proxy, consuming API quota.
**Fix**: Restrict to deployment domain(s).

### W10: DataDriver creates unnecessary Date objects + unconditional store updates

**File**: `src/components/DataDriver.tsx:15,35`
**Agent**: Complexity Analyst
**Issue**: `new Date()` created every frame when `Date.now()` suffices. `setSpacecraft` called every frame even when values haven't meaningfully changed.
**Fix**: Use `Date.now()` directly. Add dirty check (skip update if delta < threshold).
