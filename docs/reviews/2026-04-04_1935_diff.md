## Suggestions

### S1: Duplicate component trees for mobile/desktop

**File**: `src/hud/HUD.tsx:74-104`
**Agents**: Complexity Analyst, Efficiency Sentinel

MoonDistCard, DSNStatus, SpaceWeatherPanel, and CameraControls are rendered in BOTH the desktop-only `hidden sm:flex` row AND the mobile `sm:hidden` AnimatePresence section. Both sets mount to the DOM — the hidden ones are just `display:none`. Each subscribes to Zustand slices independently, doubling subscription checks.

**Suggested fix**: Use a `useMediaQuery('(min-width: 640px)')` hook for conditional rendering, or wrap all four in `memo` so hidden instances skip VDOM diffs.

---

### S2: AnimatePresence height:'auto' on mobile

**File**: `src/hud/HUD.tsx:81-96`
**Agents**: Efficiency Sentinel

`height: 'auto'` forces a layout/reflow measurement on every animation frame. On budget Android devices this could produce jank during the 0.25s toggle animation.

**Suggested fix**: Use `transform: scaleY()` with `transform-origin: top`, or measure height via ref before animating.

---

### S3: MissionEventsPanel sort not memoized

**File**: `src/hud/MissionEventsPanel.tsx:17`
**Agents**: Complexity Analyst, Efficiency Sentinel

`[...MILESTONES].sort()` re-allocates and re-sorts on every render. MILESTONES is a static 19-element config import. ProgressBar correctly wraps its identical sort in `useMemo`.

**Suggested fix**: Hoist to module scope (preferred — data never changes) or wrap in `useMemo([], [])`.

---

### S4: MissionEventsPanel SEVERITY_COLORS inside component

**File**: `src/hud/MissionEventsPanel.tsx:64`

Static constant re-created inside the function body on every render. Inconsistent with module-level constants in AlertItem (`SEVERITY_CONFIG`), SpaceWeatherPanel (`KP_COLOR`), CameraControls (`PRESETS`).

### S5: DSNStatus computation not memoized

**File**: `src/hud/DSNStatus.tsx:13-32`
